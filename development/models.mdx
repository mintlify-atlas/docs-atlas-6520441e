## Overview

The Bethlemitas platform uses Eloquent ORM to manage database interactions. Models are organized around three core domains: **Users**, **Academic Structure**, and **Psycho-orientation Services**.

## User Models

### Users_teacher

The primary user model representing teachers, psychologists, and coordinators. Extends Laravel's `Authenticatable` class.

```php
namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Spatie\Permission\Traits\HasRoles;

class Users_teacher extends Authenticatable
{
    use HasFactory, HasRoles, Notifiable;

    protected $table = 'users_teachers';
    protected $guard_name = 'web';

    protected $fillable = [
        'name',
        'lastname',
        'email',
        'password',
        'document',
        'phone',
        'id_state',
        'group_director',
    ];

    protected $hidden = [
        'password',
        'remember_token',
    ];
}
```

#### Query Scopes

Reusable query filters for common user states:

```php
// Usuarios activos
public function scopeActivos($query)
{
    return $query->where('id_state', 1);
}

// Usuarios PIAR
public function scopePiar($query)
{
    return $query->where('id_state', 2);
}

// Solo docentes, psicoorientadores o coordinador
public function scopeDocentesPsicoCoordinador($query)
{
    return $query->whereHas('roles', function ($q) {
        $q->whereIn('name', [
            'docente',
            'psicoorientador',
            'coordinador'
        ]);
    });
}
```

    ├── belongsTo Users_teacher (sent_by)

    ├── belongsTo Users_student
    └── belongsTo Users_teacher (psychologist_writes)

## Best Practices

### Eager Loading

Always eager load relationships to avoid N+1 queries:

```php
// Bad - N+1 queries
$students = Users_student::all();
foreach ($students as $student) {
    echo $student->degree->degree; // New query each iteration
}

// Good - Single query
$students = Users_student::with('degree', 'group', 'latestReferral')->get();
foreach ($students as $student) {
    echo $student->degree->degree; // No additional queries
}
```

### Using Scopes

Leverage model scopes for cleaner queries:

```php
// Instead of:
$teachers = Users_teacher::where('id_state', 1)
    ->whereHas('roles', function($q) {
        $q->where('name', 'docente');
    })->get();

// Use:
$teachers = Users_teacher::activos()->role('docente')->get();
```

### Transaction Safety

Wrap related operations in database transactions:

```php
DB::beginTransaction();
try {
    $user = new Users_student();
    $user->save();

    $referral = new Referral();
    $referral->id_user_student = $user->id;
    $referral->save();

    DB::commit();
} catch (\Exception $e) {
    DB::rollback();
    // Handle error
}
```

See app/Http/Controllers/CreateReferralController.php:90-126

<Warning>
Always use database transactions when creating related records to maintain data integrity.
</Warning>

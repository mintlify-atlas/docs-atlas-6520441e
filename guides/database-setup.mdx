## Overview

The Bethlemitas platform uses a comprehensive database schema designed for managing educational psycho-orientation services. The database includes tables for users, roles, institutions, psycho-orientation reports, and academic structure.

## Database Configuration

Configure your database connection in `.env`:

```bash .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=piarmanager
DB_USERNAME=root
DB_PASSWORD=
```

The database configuration file is located at `config/database.php`:

```php config/database.php
'mysql' => [
    'driver' => 'mysql',
    'url' => env('DB_URL'),
    'host' => env('DB_HOST', '127.0.0.1'),
    'port' => env('DB_PORT', '3306'),
    'database' => env('DB_DATABASE', 'laravel'),
    'username' => env('DB_USERNAME', 'root'),
    'password' => env('DB_PASSWORD', ''),
    'unix_socket' => env('DB_SOCKET', ''),
    'charset' => env('DB_CHARSET', 'utf8mb4'),
    'collation' => env('DB_COLLATION', 'utf8mb4_unicode_ci'),
    'prefix' => '',
    'prefix_indexes' => true,
    'strict' => true,
    'engine' => null,
],
```

## Creating the Database

<Steps>
  <Step title="Connect to MySQL">
    ```bash
    mysql -u root -p
    ```
  </Step>

  <Step title="Create Database">
    ```sql
    CREATE DATABASE piarmanager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```
  </Step>

  <Step title="Create User (Optional)">
    ```sql
    CREATE USER 'bethlemitas'@'localhost' IDENTIFIED BY 'secure_password';
    GRANT ALL PRIVILEGES ON piarmanager.* TO 'bethlemitas'@'localhost';
    FLUSH PRIVILEGES;
    ```
  </Step>

  <Step title="Verify Connection">
    ```bash
    php artisan tinker
    ```
    
    ```php
    DB::connection()->getPdo();
    ```
  </Step>
</Steps>

## Running Migrations

### Execute All Migrations

```bash
php artisan migrate
```

This runs all migrations in chronological order based on their timestamps.

### Migration Commands

<CodeGroup>
  ```bash Fresh Migration
  php artisan migrate:fresh
  ```

  ```bash Rollback Last Batch
  php artisan migrate:rollback
  ```

  ```bash Rollback All
  php artisan migrate:reset
  ```

  ```bash Refresh Database
  php artisan migrate:refresh
  ```

  ```bash Fresh with Seed
  php artisan migrate:fresh --seed
  ```
</CodeGroup>

<Warning>
  `migrate:fresh` drops all tables and recreates them. Only use in development!
</Warning>

### Psycho-Orientation System

#### Psychoorientations Table

Core table for psycho-orientation reports:

```sql
CREATE TABLE psychoorientations (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    psychologist_writes BIGINT UNSIGNED NOT NULL,
    id_user_student BIGINT UNSIGNED NOT NULL,
    age_student TINYINT NOT NULL,
    group_student VARCHAR(50) NOT NULL,
    director_group_student VARCHAR(255) NOT NULL,
    title_report VARCHAR(255) NOT NULL,
    reason_inquiry TEXT NOT NULL,
    recomendations TEXT NOT NULL,
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    
    FOREIGN KEY (psychologist_writes) 
        REFERENCES users_teachers(id),
    FOREIGN KEY (id_user_student) 
        REFERENCES users_students(id)
);
```

Defined in: `database/migrations/2024_10_02_130145_create_psychoorientations_table.php`

### Permission Tables (Spatie)

The Spatie Laravel Permission package creates these tables:

```sql
CREATE TABLE roles (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    guard_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    UNIQUE KEY roles_name_guard_name_unique (name, guard_name)
);

CREATE TABLE permissions (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    guard_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    UNIQUE KEY permissions_name_guard_name_unique (name, guard_name)
);

CREATE TABLE model_has_roles (
    role_id BIGINT UNSIGNED NOT NULL,
    model_type VARCHAR(255) NOT NULL,
    model_id BIGINT UNSIGNED NOT NULL,
    PRIMARY KEY (role_id, model_id, model_type),
    FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE
);
```

Defined in: `database/migrations/2024_10_03_133006_create_permission_tables.php`

## Database Seeders

### Running Seeders

```bash
php artisan db:seed
```

Run a specific seeder:

```bash
php artisan db:seed --class=RolSeeder
```

### Seeder Execution Order

Seeders run in this order (defined in `DatabaseSeeder.php`):

<Steps>
  <Step title="RolSeeder">
    Creates system roles and permissions:
    - `coordinador` (Coordinator)
    - `docente` (Teacher)
    - `psicoorientador` (Psycho-orientator)
    - `estudiante` (Student)
    
    Assigns `create_user` permission to coordinador, docente, and psicoorientador roles.

    ```php database/seeders/RolSeeder.php
    $coordinador = Role::create(['name' => 'coordinador']);
    $docente = Role::create(['name' => 'docente']);
    $psicoorientador = Role::create(['name' => 'psicoorientador']);
    $estudiante = Role::create(['name' => 'estudiante']);

    Permission::create(['name' => 'create_user']);

    $coordinador->givePermissionTo(['create_user']);
    $docente->givePermissionTo(['create_user']);
    $psicoorientador->givePermissionTo(['create_user']);
    ```
  </Step>

  <Step title="AreasSeeder">
    Populates academic subject areas.
  </Step>

  <Step title="DegreeSeeder">
    Creates grade levels for the institution.
  </Step>

  <Step title="GroupSeeder">
    Initializes student groups/classes.
  </Step>

  <Step title="PeriodSeeder">
    Sets up academic periods.
  </Step>

  <Step title="StateSeeder">
    Populates system status options.
  </Step>

  <Step title="UserSeeder">
    Creates initial user accounts for testing.
  </Step>

  <Step title="DeparmetSeeder">
    Adds department/region data.
  </Step>
</Steps>

### Creating Custom Seeders

Generate a new seeder:

```bash
php artisan make:seeder InstitutionSeeder
```

Example seeder structure:

```php
namespace Database\Seeders;

use Illuminate\Database\Seeder;
use App\Models\Institution;

class InstitutionSeeder extends Seeder
{
    public function run(): void
    {
        Institution::create([
            'institution' => 'Bethlemitas School',
        ]);
    }
}
```

## Database Backup and Restore

### Backup Database

```bash
mysqldump -u root -p piarmanager > backup.sql
```

### Restore Database

```bash
mysql -u root -p piarmanager < backup.sql
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Migration Already Exists Error">
    If you see "Migration already exists" errors:

    ```bash
    php artisan migrate:reset
    php artisan migrate
    ```

    Or check the migrations table:

    ```sql
    SELECT * FROM migrations;
    ```
  </Accordion>

  <Accordion title="Foreign Key Constraint Fails">
    Ensure tables are created in the correct order. Run:

    ```bash
    php artisan migrate:fresh
    ```

    This recreates all tables in the proper order.
  </Accordion>

  <Accordion title="Seeder Class Not Found">
    Clear the autoloader cache:

    ```bash
    composer dump-autoload
    php artisan db:seed
    ```
  </Accordion>

  <Accordion title="Permission Table Already Exists">
    The permission tables are created by the Spatie package. If you encounter errors:

    ```bash
    php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
    php artisan migrate:fresh
    ```
  </Accordion>
</AccordionGroup>

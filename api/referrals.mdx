## Overview

The Referrals API handles the creation and management of student referrals from teachers to psycho-orientators, as well as psychological reports and follow-ups. This is the core workflow of the Bethlemitas psycho-orientation system.

---

## Create Referral

<Card title="POST /store/referral" icon="paper-plane">
  Creates a new student referral and student record. Automatically notifies the assigned psycho-orientator via email.
</Card>

### Endpoint

```
POST /store/referral
```

### Authentication

<ParamField header="Authorization" type="session" required>
  Active session with **Docente** or **Psicoorientador** role
</ParamField>

### Request Parameters

<ParamField body="number_documment" type="string" required>
  Student's document number (1-20 digits)
</ParamField>

<ParamField body="name" type="string" required>
  Student's first name
</ParamField>

<ParamField body="last_name" type="string" required>
  Student's last name
</ParamField>

<ParamField body="degree" type="integer" required>
  Degree/grade ID (must exist in degrees table)
</ParamField>

<ParamField body="group" type="integer" required>
  Group/class ID (must exist in groups table)
</ParamField>

<ParamField body="age" type="integer" optional>
  Student's age (minimum 0)
</ParamField>

<ParamField body="reason_referral" type="string" required>
  Detailed reason for the referral
</ParamField>

<ParamField body="observation" type="string" required>
  Teacher's observations about the student
</ParamField>

<ParamField body="strategies" type="string" required>
  Strategies the teacher has already tried with the student
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X POST https://bethlemitas.example.com/store/referral \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -b "laravel_session=your_session_cookie" \
  -d "number_documment=1122334455" \
  -d "name=Juan" \
  -d "last_name=Pérez" \
  -d "degree=5" \
  -d "group=3" \
  -d "age=10" \
  -d "reason_referral=Dificultades de atención en clase" \
  -d "observation=El estudiante presenta dificultad para mantener la concentración durante las actividades académicas" \
  -d "strategies=Se ha implementado trabajo en parejas y refuerzo positivo"
```

```javascript JavaScript
const formData = new URLSearchParams({
  number_documment: '1122334455',
  name: 'Juan',
  last_name: 'Pérez',
  degree: '5',
  group: '3',
  age: '10',
  reason_referral: 'Dificultades de atención en clase',
  observation: 'El estudiante presenta dificultad para mantener la concentración',
  strategies: 'Se ha implementado trabajo en parejas y refuerzo positivo'
});

fetch('https://bethlemitas.example.com/store/referral', {
  method: 'POST',
  credentials: 'include',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: formData
})
```
</CodeGroup>

### Response

<ResponseField name="success" type="string">
  ```json
  {
    "success": "Estudiante remitido exitosamente."
  }
  ```
</ResponseField>

### Error Responses

<ResponseField name="error" type="string">
  No psycho-orientator assigned to the degree
  
  ```json
  {
    "error": "No se encontró un psicoorientador asignado para el grado seleccionado, comunicate con cordinación académica para que asigne a un psicoorientador."
  }
  ```
</ResponseField>

<ResponseField name="info" type="string">
  Student already exists and cannot be referred again
  
  ```json
  {
    "info": "El estudiante ya fue remitido y está en revisión por el psicoorientador."
  }
  ```
  
  or
  
  ```json
  {
    "info": "El estudiante ya está en proceso PIAR."
  }
  ```
  
  or
  
  ```json
  {
    "info": "El estudiante ya está en proceso DUA."
  }
  ```
</ResponseField>

### Implementation Details

- Creates student record with "activo" state and "estudiante" role
- Creates referral record linked to student and teacher
- Automatically determines assigned psycho-orientator based on degree
- Sends email notification to psycho-orientator (queued)
- Uses database transaction to ensure data consistency
- Student's `sent_by` field is set to authenticated teacher's ID
- Controller: `CreateReferralController@store_referral` (routes/web.php:142-143)

## Referral Workflow

The complete referral workflow follows these steps:

<Steps>
  <Step title="Teacher Creates Referral">
    Teacher identifies student needing support and creates referral via `POST /store/referral`
  </Step>
  
  <Step title="Automatic Assignment">
    System automatically assigns referral to psycho-orientator based on student's degree
  </Step>
  
  <Step title="Email Notification">
    Psycho-orientator receives email notification about new referral
  </Step>
  
  <Step title="Review and Update">
    Psycho-orientator reviews referral details via `GET /details/referral/{id}` and can update via `PUT /edit/details/referral/{id}`
  </Step>
  
  <Step title="Create Report">
    Psycho-orientator creates psychological report via `POST /store/report/student/{id}`, which updates student state
  </Step>
  
  <Step title="Ongoing Management">
    Student enters PIAR or DUA program. Teachers can add meeting minutes for students in these programs.
  </Step>
</Steps>

## File Storage

### PDF Annexes

Psychological report annexes are stored in the public storage:

- **Path**: `storage/app/public/annexes/student_{student_id}/`
- **Naming**: `student_{student_id}_{timestamp}.pdf`
- **Max Size**: 5MB
- **Format**: PDF only

### Accessing Files

To access a stored annex, use the path stored in the `annex_one` field:

```
https://bethlemitas.example.com/storage/{annex_one}
```

Example:
```
https://bethlemitas.example.com/storage/annexes/student_25/student_25_20260306_143022.pdf
```

## Validation Rules Summary

### Referral Creation

- `number_documment`: 1-20 digits
- `name`: Required string
- `last_name`: Required string
- `degree`: Required, must exist
- `group`: Required, must exist
- `age`: Optional integer, minimum 0
- `reason_referral`: Required string
- `observation`: Required string
- `strategies`: Required string

### Psychological Report

- All referral fields +
- `state`: Required, must be valid state ID
- `title_report`: Required string
- `reason_inquiry`: Required string
- `recomendations`: Required string
- `annex_one`: Optional PDF, max 5MB

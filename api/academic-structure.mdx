## Overview

The Academic Structure API allows coordinators to create, read, update, and delete academic groups, degrees, and subject areas. All endpoints require authentication and coordinator role.

<Note>
  All academic structure endpoints require **Coordinador** role (`RoleMiddleware`).
</Note>

---

## Groups Management

### Create Group

<Card title="POST /store/group" icon="users">
  Creates a new academic group (e.g., "10A", "11B").
</Card>

#### Endpoint

```
POST /store/group
```

#### Authentication

<ParamField header="Authorization" type="session" required>
  Active session with Coordinador role
</ParamField>

#### Request Parameters

<ParamField body="group" type="string" required>
  Group name (must be unique, e.g., "10A", "11B")
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates if the group was created successfully
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

#### Example Request

<CodeGroup>

```bash cURL
curl -X POST https://bethlemitas.edu/store/group \
  -H "Content-Type: application/json" \
  -d '{
    "group": "10A"
  }'
```

```javascript JavaScript
const response = await fetch('https://bethlemitas.edu/store/group', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    group: '10A'
  })
});

const data = await response.json();
```

</CodeGroup>

#### Implementation

From `app/Http/Controllers/CreateGroupController.php:40-51`:

```php CreateGroupController.php
public function store_group(Request $request)
{
    $request->validate([
        'group' => 'required|string|unique:groups',
    ]);

    $group = new Group();
    $group->group = $request->group;
    $group->save();

    return redirect()->back()->with('success', 'Grupo creado exitosamente.');
}
```

#### Implementation

From `app/Http/Controllers/CreateGroupController.php:54-79`:

```php CreateGroupController.php
public function update_group(Request $request)
{
    $id = $request->input('groupId');
    $group = Group::findOrFail($id);
    
    $request->validate([
        'grupo_edit' => 'required|string|unique:groups,group,' . $id,
    ]);
    
    if ($group->group === ucfirst($request->grupo_edit)) {
        return redirect()->back()->with('info', 'No hubo cambios en la actualización del grupo.');
    }
    
    $group->update([
        'group' => ucfirst($request->grupo_edit),
    ]);
    
    return redirect()->back()->with('success', 'Grupo actualizado correctamente.');
}
```

```php CreateAreaController.php
public function store_area(Request $request) {
    $request->validate([
        'area' => 'required|string|unique:areas,name_area',
    ]);

    try {
        $area = new Area();
        $area->name_area = ucfirst($request->area);
        $area->save();

        return redirect()->back()->with('success', 'Area creada exitosamente.');
    } catch (\Exception $e) {
        return redirect()->back()->with('error', 'Hubo un error al crear el area. Intente nuevamente.');
    }
}
```

## Error Responses

All endpoints may return the following error responses:

<ResponseField name="error" type="string">
  Error message describing what went wrong
</ResponseField>

### Common Errors

- **404 Not Found**: Resource not found
- **422 Unprocessable Entity**: Validation failed (duplicate name, empty fields)
- **500 Internal Server Error**: Database or server error during operation

<Warning>
  Deleting groups, degrees, or areas that have associated records (students, teachers, assignments) will fail with a foreign key constraint error. Ensure dependencies are removed first.
</Warning>

## Overview

The Students API manages student records that are created through the referral process. Students can be in different states: active, PIAR, or DUA. Both teachers and psycho-orientators can interact with student records.

---

## List Students (Teacher)

<Card title="GET /index/students/remitted" icon="list">
  Lists students referred by the authenticated teacher with "activo" state.
</Card>

### Endpoint

```
GET /index/students/remitted
```

### Authentication

<ParamField header="Authorization" type="session" required>
  Active session with **Docente** role
</ParamField>

### Query Parameters

<ParamField query="search" type="string" optional>
  Search by student name, last name, or document number
</ParamField>

<ParamField query="page" type="integer" optional>
  Page number for pagination (15 students per page)
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X GET "https://bethlemitas.example.com/index/students/remitted?search=Juan" \
  -b "laravel_session=your_session_cookie"
```

```javascript JavaScript
fetch('https://bethlemitas.example.com/index/students/remitted?search=Juan', {
  method: 'GET',
  credentials: 'include'
})
```
</CodeGroup>

### Response Schema

<ResponseField name="students" type="array">
  Paginated array of student objects
  
  <Expandable title="Student Object">
    <ResponseField name="id" type="integer">
      Student ID
    </ResponseField>
    
    <ResponseField name="number_documment" type="string">
      Student's document number
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Student's first name
    </ResponseField>
    
    <ResponseField name="last_name" type="string">
      Student's last name
    </ResponseField>
    
    <ResponseField name="age" type="integer|null">
      Student's age
    </ResponseField>
    
    <ResponseField name="id_degree" type="integer">
      Degree ID
    </ResponseField>
    
    <ResponseField name="id_group" type="integer">
      Group ID
    </ResponseField>
    
    <ResponseField name="sent_by" type="integer">
      Teacher ID who referred the student
    </ResponseField>
    
    <ResponseField name="id_state" type="integer">
      Current state ID (1 = activo, 3 = en PIAR, 4 = en DUA)
    </ResponseField>
    
    <ResponseField name="latestReferral" type="object">
      Most recent referral information
    </ResponseField>
    
    <ResponseField name="group" type="object">
      Group information
      - `id`: Group ID
      - `group`: Group name
    </ResponseField>
    
    <ResponseField name="degree" type="object">
      Degree information
      - `id`: Degree ID
      - `degree`: Degree name
    </ResponseField>
  </Expandable>
</ResponseField>

### Implementation Details

- Only returns students where `sent_by` matches authenticated teacher ID
- Filters for students with state "activo"
- Eager loads: latestReferral, group, degree relationships
- Ordered alphabetically by name and last name
- Controller: `CreateReferralController@index_student_remitted` (routes/web.php:124-125)

## Update Student

<Card title="PUT /update/student/{id}" icon="pen-to-square">
  Updates student information (basic data only, not referral details).
</Card>

### Endpoint

```
PUT /update/student/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Student ID to update
</ParamField>

### Authentication

<ParamField header="Authorization" type="session" required>
  Active session with **Docente** or **Psicoorientador** role
</ParamField>

### Request Parameters

<ParamField body="number_documment" type="string" required>
  Document number (1-20 digits, unique except for current student)
</ParamField>

<ParamField body="name" type="string" required>
  Student's first name
</ParamField>

<ParamField body="last_name" type="string" required>
  Student's last name
</ParamField>

<ParamField body="degree" type="integer" required>
  Degree ID (must exist in degrees table)
</ParamField>

<ParamField body="group" type="integer" required>
  Group ID (must exist in groups table)
</ParamField>

<ParamField body="age" type="integer" optional>
  Student's age (minimum 0)
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X PUT https://bethlemitas.example.com/update/student/25 \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -b "laravel_session=your_session_cookie" \
  -d "number_documment=1122334455" \
  -d "name=Juan Carlos" \
  -d "last_name=Pérez" \
  -d "degree=5" \
  -d "group=3" \
  -d "age=10"
```

```javascript JavaScript
const formData = new URLSearchParams({
  number_documment: '1122334455',
  name: 'Juan Carlos',
  last_name: 'Pérez',
  degree: '5',
  group: '3',
  age: '10'
});

fetch('https://bethlemitas.example.com/update/student/25', {
  method: 'PUT',
  credentials: 'include',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: formData
})
```
</CodeGroup>

### Response

<ResponseField name="success" type="string">
  ```json
  {
    "success": "Usuario editado correctamente."
  }
  ```
</ResponseField>

<ResponseField name="info" type="string">
  When no changes were detected
  
  ```json
  {
    "info": "No hubo cambios en el usuario."
  }
  ```
</ResponseField>

### Error Responses

<ResponseField name="error" type="string">
  ```json
  {
    "error": "No se encontró un psicoorientador asignado para el grado seleccionado, comunicate con cordinación académica para que asigne a un psicoorientador."
  }
  ```
</ResponseField>

### Implementation Details

- Only updates if changes are detected
- Validates that selected degree has an assigned psycho-orientator
- Does not modify referral information
- Controller: `CreateReferralController@update_student` (routes/web.php:148-149)

## Student States

Students can have the following states:

<CardGroup cols={2}>
  <Card title="Activo" icon="circle-dot">
    Initial state when referred. Pending psycho-orientator review.
  </Card>
  
  <Card title="En PIAR" icon="file-medical">
    Student accepted into PIAR (Individualized Educational Adjustments Plan) program.
  </Card>
  
  <Card title="En DUA" icon="universal-access">
    Student in DUA (Universal Design for Learning) program.
  </Card>
  
  <Card title="Suspendido" icon="ban">
    Student account suspended (cannot be referred).
  </Card>
</CardGroup>

### State Transition Rules

1. New students start with "activo" state upon referral
2. Psycho-orientator can accept students to "en PIAR" state
3. Students with "activo", "en PIAR", or "en DUA" states cannot be referred again
4. Only psycho-orientators can change student states via psychological reports
